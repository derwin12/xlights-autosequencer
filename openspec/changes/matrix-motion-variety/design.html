Matrix motion variety from mined vendor presets

Second-round fix for the repetitive 06_PROP_Matrix Spirals layer (bug-256 follow-up) — plus a verified rotation-aliasing bug fix and Lightning-as-crash-accent.

Goal

Give matrix-group placements the setting-level variety the vendor corpus actually uses (15–52 distinct combos per effect vs our 3–4), by rotating through pools of full mined parameter combinations, and fix the seed-aliasing bug that made Lightning unreachable.

Observed rotation walk in the real 1999 output: Shockwave(0) → Pinwheel(1) → Shockwave(0) → Ripple(3) → … — slot 2 (Lightning) is unreachable because idx = (variation_seed // 2) % 4 aliases with the song's qualifying-section stride. Same failure family as bug-182 and bug-188.

Design — four parts

  1. Occurrence-based rotation. place_effects counts successful corpus-recipe placements per group and passes occurrence_index to _place_corpus_recipe; rotation pools use occurrence_index % len(pool). Every slot reachable in every song. The bug-182 seed // 2 primary/alt parity for non-pool families is untouched.
  2. Mined preset pools in motion_rotation. No type change — repeated effect names with different mined params. Matrix pool grows 4 → ~7 entries (Shockwave ×3 radii/widths, Pinwheel ×2 arm-styles, Ripple ×2 implode/explode), interleaved so consecutive occurrences change effect. Lightning removed.
  3. New secondary_rotation field. Pool of parameter tuples for the sustained layer, selected per occurrence. Matrix gets 5 mined Spirals looks (thin twin-pair — the #1 vendor look — thick 3D single, fast tight flat, reverse 3D, multi-spiral). Default () keeps every other recipe bit-identical. Per-placement Flip Horizontal alternation (bug-254/256) composes on top.
  4. Lightning → crash-accent pass. _place_crash_accents adds a Lightning burst on tier-6 matrix groups at each crash mark (same window/exclusions as the whole-house Shockwave and Moving Head punch), alternating the vendor's Rotate CC 90 variant. Layer index below the group's existing minimum so it renders on top (bug-248: first EffectLayer wins).

Variability principle: variety comes from rotating complete mined combinations — never independent per-parameter sampling, which can produce combos no vendor ever shipped.

Alternatives considered

  • Per-parameter jitter from mined distributions — rejected; invents unvendor-like combos.
  • Coprime stride on the seed — rejected; still aliasable, harder to reason about than a counter.
  • Rotate the secondary effect itself (Plasma/Fire) — deferred; mined sustained-layer idiom is overwhelmingly Spirals.
  • Fix aliasing but keep Lightning in rotation — rejected; vendors use matrix Lightning as an accent (2/14 songs), not a section texture, and the user explicitly suggested the crash pairing.

Regression surface

  • PropFamilyRecipe: additive field, default (); constructed only in corpus_recipes.py.
  • _place_corpus_recipe: private, 4 call sites in effect_placer.py, no direct test callers (verified by grep) — tests go through place_effects, whose signature is unchanged and guarded by test_place_effects_signature.py.
  • _place_crash_accents: one caller (plan.py:339); additive behavior, existing assertions must keep passing.
  • Non-matrix recipes: bit-identical output (no rotation pools, empty secondary_rotation).
  • Microscope baselines: matrix output changes by design — re-run panel + matrix manifest, sensitivity gate before re-promoting.
